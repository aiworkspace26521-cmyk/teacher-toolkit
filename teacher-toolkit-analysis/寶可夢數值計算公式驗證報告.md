## 2. 所有屬性數值計算公式（完整展開）

### 2.1 核心公式（`getPokemonStats` at `kpi-dashboard.html` L2535-2555）

```
Step 1 — raw = 去除前綴（⭐/✨）與後綴（(類型)）
         leg = checkIsLegendary(raw)
         stage = EVO_STAGE_MAP[raw] ?? 0
         baseTotal = leg ? 600 : (300 + min(stage, 2) × 100)

Step 2 — primaryType = getPokemonType(baseName)[0]
         ratios = TYPE_RATIO[primaryType] ?? [1,1,1,1,1,1]
         avgBase = baseTotal / 6

Step 3 — bs_X = floor(avgBase × ratios[i])   // [HP,ATK,DEF,SPA,SPD,SPE]

Step 4 — HP  = floor(((bs_hp  × 2 + 31) × level) / 100) + level + 10
         ATK = floor(((bs_atk × 2 + 31) × level) / 100) + 5
         DEF = floor(((bs_def × 2 + 31) × level) / 100) + 5
         SPA = floor(((bs_spa × 2 + 31) × level) / 100) + 5
         SPD = floor(((bs_spd × 2 + 31) × level) / 100) + 5
         SPE = floor(((bs_spe × 2 + 31) × level) / 100) + 5

Step 5 — 傳說加成（if leg）: HP×1.2, ATK×1.3, DEF×1.2, SPA×1.3, SPD×1.2, SPE×1.15
         特殊傳說（超夢/烈空坐/阿爾宙斯/帝牙盧卡/帕路奇亞/騎拉帝納）:
         HP×1.35, ATK×1.45, DEF×1.35, SPA×1.5, SPD×1.35, SPE×1.3
```

### 3.4 找出 inject_data.js 的 Evolution Event Bug

`inject_data.js`（`D:\20260521opencode\tools\learning-kpi-dashboard\backend\inject_data.js`）第 114-122 行：

```javascript
await addEvent('Emma', 'evolution', null, {
    pokemonId: 'P0',
    speciesFrom: '伊布',
    speciesTo: '月亮伊布',
    ...
});
```

此 event 寫入 Firestore 的欄位為 `type: "evolution"`、`subType: null`。

但前端 replay（`kpi-dashboard.html` L1106 附近）讀取：
```javascript
var action = evt.action, ...
var rowAction = String(action || "");
// → 因為 evt.action = undefined，rowAction = ""（空字串）
```

而進化邏輯判斷（L1235 附近）：
```javascript
} else if (rowAction === "E") {
```

`rowAction` 為空字串 ≠ `"E"`，**進化事件被完全跳過**。

正確格式應為（參考前端 L1032-1035）：
```javascript
action: "E",
note: "進化ID:P0 => 月亮伊布 (惡系)"
```
