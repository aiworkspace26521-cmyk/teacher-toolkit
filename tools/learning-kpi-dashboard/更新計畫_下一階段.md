## 階段二：UI/UX 打磨（Priority: Medium）

### U1：行動裝置 RWD 適配（估時 ~1h）
- 現狀：max-width 520px container，但部分元素在手機上仍偏擠
- 目標：`min-width: 320px` 完整支援，按鈕/字體大小自動縮放
- 方法：media query 調整 `padding`、`font-size`、按鈕 `min-height`
- 驗證：Chrome DevTools 手機模擬 (iPhone SE / Pixel 5)

### U2：戰鬥動畫微調 + 粒子效果（估時 ~1h）
- 現狀：Task5e 已實作基本 HP/shake/flash/文字提示
- 目標：招式命中時加上 particles（火花/水花/冰晶等對應屬性）
- 方法：CSS `@keyframes` + `::before/after` pseudo-element
- 注意：不引入外部 library，維持單 HTML 檔案

### U3：Loading skeleton screen（估時 ~30m）
- 現狀：Firestore 讀取時畫面空白
- 目標：灰色脈衝動畫 skeleton 替代空白
- 方法：CSS + 切換 class `.loading` / `.loaded`

### U4：Web Audio API 音效系統（估時 ~1h）
- 現狀：完全無音效
- 目標：攻擊命中、升級、捕捉成功、進化等音效
- 方法：`AudioContext` + `OscillatorNode` 合成簡單音效（免載入外部檔案）
- 優先：攻擊命中音效 + 升級 fanfare

## 技術債

### T1：firebase.json 快取策略
- 現狀：`**/*.html` 設 `no-cache`，但 JS/CSS 未設定
- 解決：`pokemon-gen2-9.js` 加入 `Cache-Control: max-age=3600`
- 原因：檔案變動不頻繁，設快取可提升載入速度

### T2：public/ 同步自動化
- 現狀：`tools/learning-kpi-dashboard/frontend/` 與 `public/` 需手動 copy
- 解決：在 `package.json` 或 `firebase.json` 加入 predeploy script
- 範例：`"predeploy": "copy-item tools/learning-kpi-dashboard/frontend/* public/ -Force"`
