# 方案一：裝備流程閉包 Bug 修復 + PC 流程優化

> 建立日期：2026-07-08
> 相關檔案：`public/kpi-dashboard.html`

---

## 一、問題描述

### 問題一：裝備流程不合理
從 PC 點擊裝備 → 選擇寶可夢 → 裝備成功 → 跳出戰術電腦系統對話窗 → 關閉 → 跳出成就圖鑑對話窗。
使用者需關閉兩個 modal 才能回到遊戲，操作路徑過長。

### 問題二：裝備到錯誤的寶可夢（閉包 Bug）
在 PC 的「持有裝備」區點擊「裝備」按鈕 → 跳出選擇寶可夢視窗 → 點選迷你龍 →
但實際裝備到波克比（列表中的最後一隻）。**只有成就圖鑑對話窗的裝備按鈕是正確的。**

---

## 二、根因分析

### JavaScript Closure-in-Loop Bug

**位置：** `public/kpi-dashboard.html` line 6733（`openQuickEquip()` 函數內）

```javascript
// ❌ 有 bug 的寫法（原始碼）
for (var qi = 0; qi < eligible.length; qi++) {
    var p = eligible[qi];  // var 是 function scope，不是 block scope
    // ... 建立 card 元素 ...
    card.onclick = function(){
        closeModal('equipModal');
        equipHeldItem(p.id, itemId);  // p 指向同一個變數！
    };
}
```

**原因：** `var p` 是 function scope，所有 card 的 `onclick` handler **共享同一個 `p` 變數**。
當 click 事件觸發時，迴圈早已結束，`p` 的值是 `eligible` 陣列的**最後一個元素**。
因此無論點哪張卡，裝備都會跑到最後一隻寶可夢身上。

### 為什麼成就圖鑑是對的

**位置：** line 6473

```javascript
// ✅ 正確的寫法（字串插值，p.id 在建立 HTML 時就固定了）
onclick="equipHeldItem('" + p.id + "','" + _hid + "')"
```

字串插值在 `innerHTML` 賦值時就將 `p.id` 寫死進 HTML 字串中，不受 closure 影響。

## 三、修改內容

### 修改 1：閉包 Bug 修復（line 6734）

| 項目 | 說明 |
|------|------|
| 位置 | `openQuickEquip()` 內的 `card.onclick` |
| 手法 | **IIFE（Immediately Invoked Function Expression）** |
| 原理 | 每次迭代立即執行一個新函數，將 `p.id` 作為參數傳入並保存在 closure 中 |

```javascript
// ✅ 修復後的寫法
card.onclick = (function(pid){
    return function(){
        closeModal('equipModal');
        equipHeldItem(pid, itemId);  // pid 是 IIFE 參數，每次迭代獨立
    };
})(p.id);
```

### 修改 2：裝備成功後回 PC 而非成就圖鑑（line 6757）

**原始行為：** `equipHeldItem()` callback 直接呼叫 `openCollection()`，強制跳出成就圖鑑。

**新行為：** 偵測 PC modal 是否正在顯示。若是 → 呼叫 `renderPCBoxContent()` 原地刷新 PC 內容；
若不是（從成就圖鑑觸發）→ 才呼叫 `openCollection()`。

```javascript
updateDashboard();
if ($("pcBoxModal") && $("pcBoxModal").style.display === "flex") {
    renderPCBoxContent();  // 原地刷新 PC
} else {
    openCollection();      // 從圖鑑觸發時才開圖鑑
}
```

### 修改 3：卸下成功後回 PC 而非成就圖鑑（line 6769）

同修改 2 的邏輯，套用於 `unequipHeldItem()` callback。

### 修改 4：清理 `unequipFromPCBox` 多餘 reopen（line 6774-6776）

**原始程式碼：**
```javascript
function unequipFromPCBox(pokemonId, itemId) {
    unequipHeldItem(pokemonId, itemId).then(function() {
        setTimeout(function(){ openPCBoxModal(); }, 300);
    });
}
```

修改 3 的 callback 已經會自動刷新 PC，所以 `.then()` 的多餘 reopen 可以移除：

```javascript
function unequipFromPCBox(pokemonId, itemId) {
    unequipHeldItem(pokemonId, itemId);
}
```

### 修改 5：新增 📖 成就圖鑑 Nav 按鈕（line 421）

在 nav 按鈕列新增按鈕，讓使用者可手動開啟成就圖鑑：

```html
<button class="nav-btn btn-pcbox" onclick="sfx.click();openCollection()">📖 成就圖鑑</button>
```

放在「💻 電腦」按鈕之後、「📖 全國圖鑑」之前。

## 四、修改前後流程對比

### 修改前

```
PC → 持有裝備點「裝備」
  → 跳出選擇寶可夢視窗
    → 選擇寶可夢
      → ❌ 裝備到最後一隻（閉包 bug）
        → executeSave callback → 成就圖鑑跳出
          → 使用者關閉成就圖鑑
            → 使用者關閉戰術電腦
              → 完成
```

### 修改後

```
PC → 持有裝備點「裝備」
  → 跳出選擇寶可夢視窗
    → 選擇寶可夢
      → ✅ 裝備到正確的寶可夢
        → executeSave callback → PC 原地刷新（顯示 holder）
          → 完成
```

需手動開啟成就圖鑑時，點 nav 列的「📖 成就圖鑑」按鈕。
