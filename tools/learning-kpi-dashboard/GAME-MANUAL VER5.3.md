## 1. 遊戲概述

學習 KPI 管理系統將 Pokémon 遊戲化架構重建為學習管理系統，
保留 gamification 核心（等級、經驗值、成就、收集、對戰）但以「學習任務驅動」取代「戰鬥模擬」。

### 1.1 學生角色

| 角色 | 說明 |
|------|------|
| **Neil（苡逢）** | 學生 A，五年級 |
| **Emma（苡瑄）** | 學生 B，五年級 |
| **管理員** | 家長/教師，可模擬時間、發放資源、重置進度 |

### 1.2 核心資源

| 資源 | 說明 | 獲取方式 |
|------|------|---------|
| **EXP（經驗值）** | 提升寶可夢等級 | 每日任務提交、戰鬥勝利、商城道具 |
| **金幣** | 商城貨幣 | 每日任務提交、戰鬥勝利 |
| **徽章** | 道館挑戰成就 | 道館戰勝利（冷卻 ≥1 天） |
| **道具** | 戰鬥消耗品/永久強化 | 商城購買 |
| **親密度** | 影響特定進化條件 | 每日提交、捕捉、戰鬥勝利 |

## 3. 每日任務與分數系統

### 3.1 任務類別

| 類別 | 科目 | 滿分 | 任務數 |
|------|------|:----:|:-----:|
| 數理道館 | 代數、計算 | 30 | 2 |
| 國語道館 | 國學、閱讀 | 25 | 2 |
| 英文道館 | PET/FCE、閱讀 | 25 | 2 |
| 紀律與體能 | 運動、小提琴、家務、睡眠 | 20 | 4 |
| **每日滿分** | | **100** | **10** |

### 3.2 分數計算

\\\
最終分數 = (勾選任務分數總和) × 聯盟倍率

聯盟倍率：
- 一般時期：1x
- 聯盟時期（每月最後 3 天）：2x
\\\

### 3.3 提交模式

| 模式 | 選擇方式 | EXP 公式 | 金幣公式 | 效果 |
|------|---------|----------|---------|------|
| 一般提交 | 預設 | 分數 × 7 | 分數 × 2 | 全隊均衡成長 |
| 強化現有技能 | 選「強化」+ 指定寶可夢 | 分數 × 7（指定） | 分數 × 1.5 | 集中灌注指定寶可夢 |
| 挑戰新目標 | 選「捕捉」+ 選擇屬性 | 分數 × 6 | 分數 × 1.5 | 獲得新寶可夢（依分數門檻） |

### 3.4 時間系統

- 月份計算：以 2026 年 4 月為基準（第 1 月），自動遞增
- 聯盟時期：每月最後 3 天自動偵測（分數 ×2）
- 管理員可透過下拉選單模擬月份（1~10）與週狀態（一般/聯盟）

## 4. 捕捉系統

### 4.1 分數門檻與稀有度

捕捉系統有 **徽章門檻限制**：需累積足夠徽章才能遇到傳說寶可夢。

| 每日分數 | 徽章 <8 | 徽章 8~15 | 徽章 ≥16 | 捕捉等級 |
|---------|---------|----------|---------|---------|
| 0~49 | ❌ 無法捕捉 | 同左 | 同左 | — |
| 50~59 | 100% 一般 | 同左 | 同左 | Lv.6~7 (score/8) |
| 60~74 | 85% 一般 / 15% 稀有 | 同左 | 同左 | Lv.7~9 (score/8) |
| 75~94 | 65% 一般 / 35% 稀有 | 61.75% 一般 / 33.25% 稀有 / **5% 傳說** | 同左 | Lv.12~15 (score/6) |
| 95~100 | 40% 一般 / 60% 稀有 | 37.6% 一般 / 56.4% 稀有 / **6% 傳說** | 同左 | Lv.23~25 (score/4) |

### 4.2 屬性選擇
開啟「挑戰新目標」時顯示屬性下拉選單，支援 18 屬性 + 隨機。

### 4.3 捕捉流程
1. 勾選任務、選擇「挑戰新目標」
2. 選擇指定屬性（或隨機）
3. 預覽顯示可達到的稀有度等級
4. 提交後系統根據分數判定稀有度 + 從屬性池隨機選取
5. 生成寶可夢 ID、等級、雙屬性
6. 事件寫入 Firestore（action: \捕捉\）
7. Toast 通知捕捉結果

### 4.4 每日保底捕捉（P5-3b）
| 連續提交天數 | 保底獎勵 |
|:-----------:|---------|
| 5 天 | 手動選擇一隻**稀有級**寶可夢 |
| 10 天 | 50% 傳說 / 50% 稀有（隨機抽選） |
- 使用 Firestore 追蹤連續提交天數（submitStreak）
- 儀表板顯示火焰動畫 + 預覽保底提示
- 需每日確實提交任務才計入連續天數

### 4.5 MIN 補償（P7-3）
長期提交但少戰鬥的玩家自動獲得徽章補償（在 recalculateStudentState 中計算）：
| 總提交天數 | 自動獲得徽章 |
|:---------:|:-----------:|
| ≥30 天 | +2 徽章 |
| ≥60 天 | +5 徽章 |

## 9. 戰鬥系統

### 9.1 戰鬥流程
\\\
回合制輪流攻擊（速度高者先攻）
  ├─ 天氣效果（回合開始判定）
  ├─ 特性觸發（出場/回合開始）
  ├─ 狀態檢查（凍結/睡眠/麻痺）
  ├─ 招式選擇（4 選 1）
  ├─ 傷害計算（屬性相剋 + 能力階級 + 道具）
  ├─ 狀態/異常判定
  ├─ 持續傷害（燒傷/中毒/寄生種子）
  └─ 切換下一隻（若當前昏厥）
\\\

### 9.2 傷害公式
\\\
物理：baseDamage = floor(((Lv×2/5+2)×power×atkStat/defStat)/50+2)
特殊：baseDamage = floor(((Lv×2/5+2)×power×spatkStat/spdefStat)/50+2)

修正（依序疊乘）：
1. 天氣（雨/晴/沙/雹）
2. 特性（猛火/激流等）
3. 適應力 STAB ×1.33
4. 道具（達人帶×1.2、生命寶珠×1.3、AV背心×1.5）
5. 傳說互打 ×1.5
6. 屬性相剋 ×0/0.5/1/2
7. 最小值 max(1, dmg)
8. 隨機 ×0.85~1.0
9. 會心（6.25%，焦點鏡 12.5%）×1.5
10. 最終 min(計算值, 剩餘 HP)
\\\

### 9.3 天氣系統
大晴天：火×1.5、水×2/3、日光束/日光刃×1.5（免充能）。下雨：水×1.5、火×2/3、日光束/日光刃×0.5。沙暴：岩特防×1.5。冰雹：冰×1.5。持續 5 回合。

### 9.4 能力變化
每階 ×1.1^n，範圍 -6~+6。劍舞 ATK+2、冥想 SPA+SPD+1、龍之舞 ATK+SPE+1。

### 9.5 狀態異常
燒傷（6.25%/回）、麻痺（25% 無法行動）、凍結（完全不能動）、中毒（持續傷害）、睡眠（50% 甦醒）、混亂（50% 自傷）、寄生種子（8.33%/回）

## 10. 路人戰與道館戰

### 10.1 路人戰（野外遭遇）

| 項目 | 數值 |
|------|------|
| 每日次數上限 | 5 次 |
| 遭遇分佈 | 一般 ~75% / 稀有 ~24% / 傳說 **1%** |
| 等級公式 | 訓練家等級 + bonus（傳說+30%、稀有+20%）+ 地區調整 |
| 地區調整 | 關都 -5、城都 ±0、合眾 +5、卡洛斯 +10 |
| 地區池 | 各 ~50 種非傳說常見物種（curated REGION_COMMON_POOLS） |
| 戰鬥獎勵 | EXP = 80 + 敵等級 × 3 |
| 捕獲 | ❌ 不可 |

### 10.2 簡單模式（P5-3a）
- 儀表板 checkbox 切換，Firestore 持久化
- 敵方等級 ×0.75（適用路人/道館/聯盟/魔王/衛冕）
- 戰鬥標題顯示 🌱 標記
- 60 分以下學生推薦開啟

### 10.3 32 道館戰

| 地區 | 道館數 | 波次 | 等級加成 |
|:----:|:------:|:----:|:--------:|
| 關都 | 8 | 3~4 波 | Lv.0~14 |
| 城都 | 8 | 4~5 波 | Lv.17~38 |
| 合眾 | 8 | 4~5 波 | Lv.48~75 |
| 卡洛斯 | 8 | 4~5 波 | Lv.55~88 |

戰鬥為**多波次制**，每波獨立隊伍，最後一波為館主。徽章 14~16 經平滑化調整（lvBonus 32/36/38），確保 MIN 學生也能在 M6 前達標。通關獎勵：
- EXP 加總所有波次（最後一波 ×2.5）
- 金幣 15 + 敵等級 ×0.8
- 徽章 +1（冷卻 ≥1 天）
- 15% 捕獲率

### 10.4 戰鬥隊伍
- 自動使用 PC Box 設定的隊伍（最多 6 隻）
- 戰鬥中可切換、可使用道具（藥水/活力塊/糖果）
- 攜帶道具在戰鬥中自動生效

## 21. 資料架構

\\\
events/{eventId}             → 事件溯源（不可變日誌）
students/{studentId}         → 學生當前狀態（由事件重算）
subjects/{subjectId}         → 科目定義（任務模板）
trade_offers/{offerId}       → 交換請求（暫存）
\\\

所有操作採用**事件溯源（Event Sourcing）**模式：
- 資料不可變：只新增、不修改/刪除
- 狀態由事件重算（\ecalculateStudentState\）
- 前端使用 Firestore 即時同步 + sessionStorage 快取
- 批量寫入合併（500ms debounce）

## 25. 開發者指引（§25）

### 本地開發

```bash
# 1. 模擬測試（無需 Firebase）
node tools/learning-kpi-dashboard/test/simulation-test.js

# 2. E2E 測試（需部署至 Firebase 或本地伺服器）
npm run test:e2e

# 3. 快速模擬測試
npm run test:sim
```

### 專案結構

```
tools/learning-kpi-dashboard/
├── frontend/
│   └── kpi-dashboard.html    # 主前端（單一 HTML，含 JS+CSS）
├── backend/
│   ├── index.js               # Firebase Cloud Functions
│   └── package.json
├── firestore/
│   ├── firestore.rules        # 安全規則
│   └── firestore.indexes.json # 複合索引
├── test/
│   ├── simulation-test.js     # 9 個月模擬測試（Node.js）
│   └── e2e/
│       └── smoke.spec.js      # Playwright E2E 測試
├── GAME-MANUAL.md             # 遊戲系統說明書
├── P9更新計畫.md               # 當前迭代計畫
└── 其他更新計畫.md
```

### 部署流程

1. 修改 `frontend/kpi-dashboard.html`
2. 同步修改 `public/kpi-dashboard.html`（Firebase Hosting 部署版本）
3. 執行模擬測試確認無回歸
4. 提交 PR → CI 自動執行測試
5. 合併至 master → 自動部署至 GitHub Pages + Firebase Hosting

### 重要約定

- **雙檔案同步**：`tools/learning-kpi-dashboard/frontend/kpi-dashboard.html` 與 `public/kpi-dashboard.html` 必須保持一致
- **模擬測試獨立常數**：`simulation-test.js` 有自己的常數副本（MOVE_DATABASE、GYM_LEADERS 等），前端修改後需同步更新
- **事件溯源**：學生狀態由 `kpi_events` 日誌重新計算，不可直接修改 `kpi_students` 欄位
- **去識別化**：學生資料使用座號，不使用真實姓名

### 常見 CI 檢查

| 檢查項目 | 觸發條件 | 所需時間 |
|---------|---------|---------|
| 模擬測試 | 每次 push | ~15s |
| E2E 測試 | 每次 push | ~60s |
| GAME-MANUAL 一致性 | 每次 push | ~5s |
| Firestore 規則/索引檢查 | 每次 push | ~5s |
