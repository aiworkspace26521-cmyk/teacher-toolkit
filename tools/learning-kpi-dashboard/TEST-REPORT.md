## 三、Bug 修復記錄

### Bug #1：登入下拉選單 ID 不匹配 (Critical)
- **位置**：HTML `#studentSelect` vs JS `$("studentId")`
- **症狀**：點選學生名稱無法觸發登入
- **原因**：V1.5 合併時 JS 與 HTML 採用不同 ID
- **修復**：JS 全部改為引用 `$("studentSelect")`

### Bug #2：任務分數永遠為 0 (Critical)
- **位置**：`updatePreview()` 讀取 `cb.dataset.points` 但 HTML 未設定
- **症狀**：勾選任務後分數為 0，無法提交
- **原因**：HTML 使用 `value` 屬性，JS 使用 `dataset.points`
- **修復**：HTML 全部加上 `data-points` 和 `data-label` 屬性

### Bug #3：PvP 雙倍傷害 (Major)
- **位置**：`executeSinglePvPAtk()` 第 1588 行
- **症狀**：PvP 中 `executeTurn()` 已扣血，呼叫後又重複扣一次
- **修復**：移除重複的 `defender.currentHp -= result.dmg`

### Bug #4：Admin 操作未寫入 Firestore (Major)
- **位置**：`devAdd()` 
- **症狀**：管理員加幣/加徽章只改前端 `globalData`，重整後消失
- **修復**：改為非同步 `executeSave()` 寫入 Firestore

### Bug #5：道具消耗未寫入資料庫 (Major)
- **位置**：`useItemInArena()`
- **症狀**：戰鬥中使用藥水/糖果，頁面重整後數量回復
- **修復**：加入 `executeSave()` 記錄消耗事件

### Bug #6：`leagueBadge` 元素不存在 (Minor)
- **位置**：`updatePreview()` 引用 `$("leagueBadge")`
- **症狀**：勾選任務時 JS 報錯，preview 不更新
- **修復**：移除對不存在的 DOM 元素的引用

### Bug #7：Shop 元素 ID 不匹配 (Minor)
- **位置**：`openShop()` 使用 `$("shopItems")` 但 HTML 為 `shopList`
- **症狀**：打開商城時報錯
- **修復**：JS 改為引用 `$("shopList")`

### Bug #8：Battle button ID 不匹配 (Minor)
- **位置**：`updateBattleButtons()` 使用 `$("arenaBattleBtn")`
- **症狀**：登入後 JS 報錯，Dashboard 不顯示
- **修復**：路人戰按鈕 ID 改為 `arenaBattleBtn`
